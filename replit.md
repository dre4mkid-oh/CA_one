# WorkFlow — Work Time Tracker

## Overview

A personal work time tracker built as a React + Vite web app in a pnpm monorepo with an Express API server and PostgreSQL database.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec at `lib/api-spec/openapi.yaml`)
- **Frontend**: React + Vite + React Router + TanStack Query
- **Styling**: Tailwind CSS + shadcn/ui

## Application Features

- **Today Page** (`/`): Live clock, 3 daily non-negotiable tasks with completion checkboxes, streak counter (with restore mode for missed days), task log section with start/end times and Block 1–4 checkboxes
- **Stats Page** (`/stats`): Activity heatmap (consistency map), streak stats, completion rates (weekly/monthly), total task log entries
- **Countdown Page** (`/countdown`): Fixed 11:11 PM power-down and 8 AM wake-up countdowns, plus 3 custom event countdowns (stored in DB)
- **Theme toggles**: Neon/pastel color palette toggle and light/dark mode toggle

## Project Structure

```
artifacts/
  api-server/         — Express API server (@workspace/api-server)
    src/routes/       — Route handlers: daily-tasks, task-log, streaks, countdowns, stats
    src/lib/          — Shared logic: streaks.ts (streak calculation + restore mode)
  workflow-tracker/   — React + Vite frontend (@workspace/workflow-tracker)
    src/pages/        — today.tsx, stats.tsx, countdown.tsx, layout.tsx, not-found.tsx
    src/components/   — UI components (shadcn/ui)
    src/hooks/        — use-timer.ts (live clock hook)
lib/
  api-spec/           — OpenAPI YAML spec + Orval config (generates API client)
  api-zod/            — Zod schemas generated from OpenAPI spec (@workspace/api-zod)
  api-client-react/   — React Query hooks generated by Orval (@workspace/api-client-react)
  db/                 — Drizzle ORM schema + DB connection (@workspace/db)
    schema.ts         — Tables: daily_tasks, task_log, streaks, countdowns
```

## Database Tables

- `daily_tasks` — 3 daily tasks per date (position 1-3, taskText, completed, completedAt)
- `task_log` — Time log entries (startTime, endTime, block1-4 checkboxes)
- `streaks` — Current streak, longest streak, restore mode tracking
- `countdowns` — Custom countdown events (label, targetDate)

## Streak System

- Missing 1 day → **restore mode** (need to complete 6 tasks in one day to restore streak)
- Missing 2+ days → streak resets to 0
- Logic in `artifacts/api-server/src/lib/streaks.ts`

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally
- `pnpm --filter @workspace/workflow-tracker run dev` — run frontend locally

## Important Notes

- **Date query params**: Orval-generated hooks pass date as string; do NOT use Zod `date()` for query param validation in routes. Extract from `req.query` manually.
- **Codegen barrel fix**: After running orval, a post-codegen script rewrites `lib/api-zod/src/index.ts` to only export from `"./generated/api"` to avoid duplicate export conflicts. Do NOT revert this.
- **Streak auto-init**: On each request to `/api/daily-tasks`, a `getOrCreateStreak()` call ensures the streak row exists before any comparisons.
- **Task auto-fill**: The Today page auto-creates 3 empty task rows on first load using a ref-guarded `useEffect` (one-shot per date) to prevent infinite loops.
